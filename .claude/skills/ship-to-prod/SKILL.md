# Выкат на прод

Дёргать ПЕРЕД любым выкатом в `main` (push = автодеплой на прод).
Главный закон: **зелёно локально ≠ работает в проде.** Тесты/сборка необходимы, но НЕ доказывают прод.

## Шаги
1. **Сам прочитай дифф** субагента (verify, then trust — сводка ≠ факт).
2. **Гейт сборки:** `pnpm turbo build` (строже tsc; хук `pre-push-build-gate.sh` всё равно заблокирует без свежей сборки).
3. **Ревью:** `code-reviewer` на пачку изменений; `security-reviewer` для auth/секретов/денег/доступа.
4. **🔴 Прод-проверка реальным браузером — `release-qa`.** Перед словом «готово». Не curl.
   - Особое внимание классу «расхождение сред»: build-time `NEXT_PUBLIC_*`, контракт перёд↔зад,
     сериализация/глобальные изменения (инцидент 2026-06-18: глобальный Zod-serializer уронил API 500).
   - Если прокликать нельзя (нет тест-входов на проде) — фича **НЕ готова**; говорим заказчику
     «нужна твоя ручная проверка вот по шагам», а не объявляем готовым.
5. **Рискованные инфра/контракт/safety-механизмы — сперва на staging, потом на прод.** Новый механизм
   (откат, гейт) не включаем на проде непроверенным.
6. **Батчить деплои:** не пушить по 4-5 раз в час; каждый push = окно передеплоя.
7. **Подтвердить поведением на проде** (health 200 + ключевой флоу), потом «готово» с доказательством.

## Гейт (код/CI — нельзя пропустить, не зависит от моей памяти)
- Хук `pre-push-build-gate.sh` блокирует push без свежей сборки.
- Деплой `vps-deploy.yml`: health-проверка после выката (ловит крашлуп/500).
- (доводится в #174) блокирующий smoke браузером + откат на ПОДТВЕРЖДЁННО-рабочий SHA.
- Связано: канон §0–§2, роль `release-qa`.
