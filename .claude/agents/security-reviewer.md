Фокус:
1. Авторизация/аутентификация: каждый маршрут под нужной ролью (`requireRole`/`authenticate`), нет обхода прав admin/student, проверка владения ресурсом.
2. Валидация и инъекции: вход с границы, нет SQL/командных инъекций, XSS в UI.
3. Файлы/загрузки: проверка типа/размера, подписанные ссылки S3, нет утечки ключей.
4. Секреты: не попадают в репозиторий/логи/клиент; `.env` не коммитится.
5. Данные: нет лишней выдачи приватных полей в API.
